## 1. Hash Set (set)

### 개념

중복 없는 원소들의 집합. 순서 없음. **O(1) 평균** 탐색/삽입/삭제.

언제 쓰나: "이미 본 값인지 확인해야 할 때", "중복 제거"

### 그림으로 보기

```
set = {1, 3, 5, 7}

   ┌───┬───┬───┬───┐
   │ 1 │ 3 │ 5 │ 7 │   순서 없음, 중복 없음
   └───┴───┴───┴───┘

체크: 5가 있나? → O(1)에 즉시 답
```

### 함수 사용법

```python
# 생성
s = set()                    # 빈 set
s = {1, 2, 3}                # 초기값 있음
s = set([1, 2, 2, 3])        # 리스트에서 → {1, 2, 3} (중복 제거)

# 추가
s.add(4)                     # {1, 2, 3, 4}

# 삭제
s.remove(2)                  # 없으면 에러
s.discard(2)                 # 없어도 에러 안 남 (안전)

# 조회 (가장 많이 씀)
if 3 in s:                   # O(1)
    print("found")

# 크기
len(s)

# 순회
for x in s:
    print(x)

# 집합 연산
a = {1, 2, 3}
b = {2, 3, 4}
a | b                        # 합집합 {1, 2, 3, 4}
a & b                        # 교집합 {2, 3}
a - b                        # 차집합 {1}
```

### 대표 LeetCode 문제

```python
# Contains Duplicate (217)
def containsDuplicate(nums):
    seen = set()
    for num in nums:
        if num in seen:      # O(1) 체크
            return True
        seen.add(num)
    return False
```

## 2. Hash Map (dict)

### 개념

키-값 쌍 저장. 키는 unique. **O(1) 평균** 탐색/삽입/삭제.

언제 쓰나: "값에 추가 정보를 매핑할 때" (인덱스, 카운트, 그룹 등)

### 그림으로 보기

```
dict = {"apple": 3, "banana": 5, "cherry": 2}

   ┌─────────┬───┐
   │ "apple" │ 3 │
   ├─────────┼───┤
   │ "banana"│ 5 │
   ├─────────┼───┤
   │ "cherry"│ 2 │
   └─────────┴───┘

  Key       Value
  
조회: "banana"의 값? → O(1)에 5
```

### 함수 사용법

```python
# 생성
d = {}                       # 빈 dict
d = {"a": 1, "b": 2}         # 초기값
d = dict()                   # 빈 dict

# 추가/수정
d["c"] = 3                   # 추가
d["a"] = 10                  # 수정

# 삭제
del d["a"]                   # 없으면 에러
d.pop("a", None)             # 없어도 에러 안 남 (None 반환)

# 조회 (가장 많이 씀)
if "a" in d:                 # 키 존재 체크 O(1)
    print(d["a"])

d.get("a", 0)                # 없으면 default(0) 반환 → 안전

# 카운팅 패턴 (매우 자주 씀)
counts = {}
for x in nums:
    counts[x] = counts.get(x, 0) + 1
# 또는:
from collections import defaultdict
counts = defaultdict(int)
for x in nums:
    counts[x] += 1           # 자동으로 0부터 시작

# 순회
for key in d:                # 키만
    print(key)
for key, val in d.items():   # 키와 값 같이
    print(key, val)
for val in d.values():       # 값만
    print(val)

# 크기
len(d)
```

### 대표 LeetCode 문제

```python
# Two Sum (1) - 해시맵 활용의 정석
def twoSum(nums, target):
    seen = {}                # value → index 매핑
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

# Valid Anagram (242) - 카운팅 패턴
def isAnagram(s, t):
    if len(s) != len(t):
        return False
    counts = {}
    for c in s:
        counts[c] = counts.get(c, 0) + 1
    for c in t:
        if c not in counts or counts[c] == 0:
            return False
        counts[c] -= 1
    return True
```

## 3. Stack (list로 구현)

### 개념

LIFO (Last In, First Out). 마지막에 넣은 게 먼저 나옴. **O(1)** push/pop.

언제 쓰나: "괄호 매칭", "되돌아가야 할 때", "DFS"

### 그림으로 보기

```
push(1) push(2) push(3)        pop() → 3        pop() → 2

  ┌───┐                          ┌───┐             ┌───┐
  │ 3 │ ← top                    │   │             │   │
  ├───┤                          ├───┤             ├───┤
  │ 2 │                          │ 2 │ ← top       │   │
  ├───┤                          ├───┤             ├───┤
  │ 1 │                          │ 1 │             │ 1 │ ← top
  └───┘                          └───┘             └───┘

마지막에 들어간 게 가장 먼저 나옴
```

### 함수 사용법

```python
# 생성 (그냥 list 씀)
stack = []

# Push (맨 뒤에 추가)
stack.append(1)              # [1]
stack.append(2)              # [1, 2]
stack.append(3)              # [1, 2, 3]

# Pop (맨 뒤에서 꺼냄)
top = stack.pop()            # top = 3, stack = [1, 2]

# Peek (꺼내지 않고 보기)
top = stack[-1]              # 가장 위 원소

# 비었는지 체크
if not stack:                # 빈 리스트는 falsy
    print("empty")

# 크기
len(stack)
```

### 대표 LeetCode 문제

```python
# Valid Parentheses (20) - 스택의 정석
def isValid(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    
    for c in s:
        if c in '([{':           # 여는 괄호면 push
            stack.append(c)
        else:                    # 닫는 괄호면 pop해서 매칭 확인
            if not stack or stack.pop() != pairs[c]:
                return False
    
    return len(stack) == 0       # 다 매칭됐으면 비어 있어야 함
```

## 4. Queue (collections.deque)

### 개념

FIFO (First In, First Out). 먼저 넣은 게 먼저 나옴. **O(1)** 양쪽 추가/삭제.

언제 쓰나: "BFS", "순서대로 처리해야 할 때"

⚠️ 일반 list로 queue 만들지 마세요. `pop(0)`이 O(n)이라 느림. 반드시 `deque` 사용.

### 그림으로 보기

```
enqueue(1, 2, 3)                dequeue() → 1

front                            front
  ↓                               ↓
  ┌───┬───┬───┐                  ┌───┬───┐
  │ 1 │ 2 │ 3 │                  │ 2 │ 3 │
  └───┴───┴───┘                  └───┴───┘
              ↑                            ↑
            back                         back

먼저 들어간 게 가장 먼저 나옴
```

### 함수 사용법

```python
from collections import deque

# 생성
q = deque()                  # 빈 큐
q = deque([1, 2, 3])         # 초기값

# Enqueue (뒤에 추가)
q.append(4)                  # deque([1, 2, 3, 4])

# Dequeue (앞에서 꺼냄)
front = q.popleft()          # front = 1, q = deque([2, 3, 4])

# Peek (앞에서 보기만)
front = q[0]

# 비었는지 체크
if not q:
    print("empty")

# 크기
len(q)

# 양쪽에서 다 가능 (deque의 장점)
q.appendleft(0)              # 앞에 추가
q.pop()                      # 뒤에서 꺼냄
```

### 대표 LeetCode 문제

```python
# Binary Tree Level Order Traversal (102) - BFS의 정석
from collections import deque

def levelOrder(root):
    if not root:
        return []
    
    result = []
    q = deque([root])
    
    while q:
        level = []
        for _ in range(len(q)):    # 현재 레벨 크기만큼만 처리
            node = q.popleft()
            level.append(node.val)
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
        result.append(level)
    
    return result
```

## 5. 자주 쓰는 추가 자료구조

### Counter (카운팅 전용 dict)

```python
from collections import Counter

# 생성과 동시에 카운팅
c = Counter("aabbbc")        # Counter({'b': 3, 'a': 2, 'c': 1})
c = Counter([1, 1, 2, 3])    # Counter({1: 2, 2: 1, 3: 1})

# 자주 쓰는 메서드
c.most_common(2)             # 상위 2개 [('b', 3), ('a', 2)]
c['a']                       # 2 (없으면 0 반환, 에러 안 남)
```

### defaultdict (기본값 있는 dict)

```python
from collections import defaultdict

# int 기본값 (카운팅용)
d = defaultdict(int)
d['a'] += 1                  # 자동으로 0부터 시작 → {'a': 1}

# list 기본값 (그룹핑용)
d = defaultdict(list)
d['fruit'].append('apple')   # 자동으로 빈 리스트 생성 → {'fruit': ['apple']}
```

### Heap (우선순위 큐)

```python
import heapq

# 최소 힙만 지원 (작은 게 먼저 나옴)
h = []
heapq.heappush(h, 3)
heapq.heappush(h, 1)
heapq.heappush(h, 5)
heapq.heappop(h)             # 1 (가장 작은 값)

# 최대 힙은 음수로 트릭
heapq.heappush(h, -3)        # 3을 최대 힙처럼 쓰려면 -3 저장
-heapq.heappop(h)            # 꺼낼 때 다시 부호 뒤집기
```

## 6. 시간복잡도 한눈에

| 자료구조 | 검색 | 삽입 | 삭제 |
|---------|------|------|------|
| set     | O(1) | O(1) | O(1) |
| dict    | O(1) | O(1) | O(1) |
| stack (list) | O(n) | O(1) | O(1) |
| queue (deque) | O(n) | O(1) | O(1) |
| list    | O(n) | O(1) 끝, O(n) 중간 | O(n) |

⚠️ list.pop(0)는 O(n) — Queue 용도로 쓰지 마세요. deque 쓰세요.
